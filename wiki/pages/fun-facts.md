# Fun Facts (AI)

Short Spanish-language trivia paragraphs about albums and songs, generated by LLMs and cached persistently in `fun-facts-db.json`.

## Cache key

`"album title lowercase::artist name lowercase"` — e.g. `"abbey road::the beatles"`.

Each cache entry is an array of up to **15** fact strings. Facts are in Spanish; artist/album/song names stay in English.

## Fetch flow

```
GET /api/fun-facts?album=...&artist=...
        │
        ▼
  15+ facts in cache?
    ├── Yes → return immediately
    └── No  → fetch_fun_facts(title, artist) [blocking]
                    │
                    ▼
              Try Gemini first
                    │
              Gemini fails / empty?
                    └── Try Groq
                    │
              Parse response → append to cache
              Write fun-facts-db.json
              Return facts
```

## AI providers

| Provider | Model | Rate limit handling |
|----------|-------|-------------------|
| Gemini | `gemini-2.0-flash-lite` | `_gemini_rate_lock` enforces ≥ 5s between calls (≤ 12/min, safely under the 15/min API limit) |
| Groq | `llama-3.3-70b-versatile` (or configured) | Fallback; used if Gemini unavailable or returns empty |

Both use `urllib.request` (no external HTTP library). API keys from `.env`: `GEMINI_API_KEY`, `GROQ_API_KEY`.

## Background prefetch worker — `_prefetch_worker`

A daemon thread started **6 seconds after boot** (delay lets the server stabilize):

1. Check `_priority_album` — if set, fetch its facts first. This is set whenever the admin sets now-playing, so the currently-playing album always gets facts loaded quickly.
2. Iterate all albums in `discogs-collection.json`.
3. For any album with < 15 cached facts, call `fetch_fun_facts`.
4. Sleep 120 seconds, then repeat.

The priority mechanism ensures that what's currently playing has facts available even before the full collection is prefetched.

## In-memory cache

`_fun_facts_db` is loaded from disk at startup and kept in sync as facts are added. The `_fun_facts_lock` guards access to this dict from both the prefetch worker thread and request handler threads.

## Graceful degradation

If both API keys are missing or both providers fail, `GET /api/fun-facts` returns an empty array `[]`. The server starts and operates normally without any AI keys.

## Prompt

The prompt asks the model for 15 facts per album, covering:
- Album history and recording context
- Individual song facts
- Music video facts
- All written in Spanish (names stay in English)
