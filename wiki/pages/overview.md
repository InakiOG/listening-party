# Project Overview

**Listening Party** is a local-network web app for hosting vinyl listening parties. A group of people in the same room join from their phones, browse the host's Discogs vinyl collection, and leave star ratings and text reviews for albums and songs in real time. The host (the `administrador` account) controls what's currently playing.

## Design philosophy

The core constraint is **zero external runtime dependencies**. No npm, no pip packages beyond pytest for testing, no database, no build step. Everything runs on Python's standard library and vanilla JavaScript. This makes it trivially portable — clone, run `python server.py`, done.

This constraint shapes every layer:
- Storage is flat JSON files, not a database.
- The HTTP server is `http.server.ThreadingHTTPServer`, not Flask or FastAPI.
- The frontend is a hand-written SPA, not React or Vue.

## Scope

The app is explicitly designed for **trusted guests on a private local network**. Security trade-offs (plaintext passwords, no HTTPS, no CSRF protection) are intentional, not oversights. See [[security-notes]] for the full list.

## Key actors

| Actor | Account name | What they can do |
|-------|-------------|-----------------|
| Guest | `usuario` | Browse collection, submit reviews, like reviews, update profile |
| Host | `administrador` | Everything guests can do, plus: set now-playing, clear now-playing, finish party, add live albums, attach party photo |

## What makes it interesting technically

- **Thread safety without a database**: two locks (`REVIEWS_LOCK`, `LIVE_ALBUMS_LOCK`) guard all JSON file reads and writes. See [[thread-safety]].
- **Physics-based UI**: active user profile photos animate as bouncing bubbles on a canvas. See [[frontend]].
- **AI fun facts**: Spanish-language trivia about albums is generated by Gemini or Groq and cached persistently. See [[fun-facts]].
- **Discogs sync**: the vinyl collection is fetched from the Discogs API and cached locally, with a background thread backfilling missing track lists from MusicBrainz and iTunes. See [[discogs-integration]].
